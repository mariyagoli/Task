• git config --global user.name - додаємо ім'я користувача
  • git config --global user.email - додаємо e-mail користувача
  • git init - створення нового GIT репозиторію
  • git status - відображає зміни у каталозі, показує файли, що не відстежуються
  • git add - додає зміни у Stage Area
  • git commit - команда, за якою зберігатимуться зміни на локальному репозиторії
  • git log - показує історію коммітів
  • git chekout - перехід на гілку та оновлення каталогу
  • git branch - створює нову гілку
  • git push - завантеження локальних коммітів до GitHUb
  • git pull - оновлює усі локальні робочі гілки з новими коммітами та відправляє зміни до GitHub
  • git merge - поєднує віддалену гілку з поточною локальною
  • git clone [url] -скачування відділеного репозиторію репозиторію з файлами та коммітами
  • .gitignore - потрібен для приховування файлів та папок від системи контролю версій
  • git revert -скасовує комміт, шляхом створення нового коміту
  • git reset - використовується для відміни змін
  • git rebase - варіант інтегрування змін з однієї гілки в іншу
  • git сherry-pick - перенос окремих коммітів між гілками
  • git stash - зміни приховуються в файлах, зберігаються зміни окремо, їх можна повернути
